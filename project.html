    The Final Project Information

    Dihong Gong
    No unread replies. No replies.

    Dear Students,

    The final project is due by Thursday April 28th at noon. A project pdf should be submitted via Canvas) with the subject title Final project: CAP6610 Spring 2016 and attachment entitled:

    cap6610sp16_project_final_<insert team member last names separated by _ >.pdf.

    You need to include all the source codes of the algorithms that you have implemented on your own.

    If the file is too big and/or you want to send us an archive, please place the archive on google drive, dropbox, skydrive or spideroak and send us a direct link.

    If the project team wishes to show us a demo, they can do so by setting up an appointment with the TAs. This has to be done before noon on Thursday, April 28th.

    Please read the uploaded paper in Files on an empirical comparison of supervised learningalgorithms and use the UCI Machine Learning repository for data sets. The paper should give you an idea of how to run supervised learning algorithms. You don't have to follow every procedure in it.

    Minimal requirements for the final submission:

    Algorithms: Support vector machines, Random forests, Deep learning and your own implementation from scratch of the multi-class kernel SVM.

    Datasets: Breast Cancer Wisconsin (Original), Optical recognition of handwritten digits, Forest type mapping.

    For each dataset, the team should separate out training and testing patterns. The ratios attempted for training and testing should range from 10% training/90% testing to 50% training/50% testing. For the sake of cross validation, training sets can be further subdivided into training and validation sets (typical divisions range from 70/30 to 90/10 percentages) and use k-fold cross-validation.

    You must explain the workings of each algorithm and not merely treat it as a black box. Summarize how the algorithms' performance changes as free parameters are varied. Do not just mindlessly runlibsvm.so and expect to receive a lot of credit. In addition, you must write up in full detail, your implementation of the multi-class kernel SVM.

    **The project will be competitively graded so each team is encouraged to go beyond using just the above datasets and algorithms.

    Thanks,

    Dihong
